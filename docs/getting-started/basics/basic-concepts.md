# 基础概念

这一页用新手语言解释 Arqel 文档里反复出现的几个词。先理解这些词，再去复制命令会更安全。

## API 是什么

API 可以理解成“软件之间说话的接口”。

你平时在网页里点击按钮，是人在操作网页。API 则是程序用固定格式向另一个服务发送请求，比如：

- 你的代码向 Arqel 发送一段文字。
- Arqel 把请求转给对应模型服务。
- 模型返回回答。
- Arqel 再把回答返回给你的代码或工具。

## 请求是什么

一次请求就是一次“发消息给服务”。

一个请求通常包含：

- 请求地址：发到哪里。
- 请求头：证明身份、说明数据格式。
- 请求体：你真正想让模型处理的内容。
- 模型名：这次要调用哪个具体模型。

## API Key 是什么

API Key 是你的调用凭证，类似一把钥匙。

它用来告诉 Arqel：这次请求属于哪个账户、是否有权限、用量应该记到哪里。

API Key 不能公开：

- 不要发到群聊。
- 不要写进公开 GitHub 仓库。
- 不要放进浏览器前端代码。
- 不要贴给客服完整 Key，排查时最多提供前后几位。

## Base URL 是什么

Base URL 是客户端要请求的 API 根地址。

如果把一次请求比作寄快递：

- Base URL 是收件地址的大楼。
- `/chat/completions` 是楼里的具体窗口。
- API Key 是你的身份证明。
- 模型名是你要找的具体服务人员。

示例：

```text
https://api.arqel.dev/v1
```

实际地址以 Arqel 控制台为准。

## 模型名是什么

模型名决定这次请求使用哪个具体模型。

模型家族名和具体模型名不是一回事。例如“Claude”“GPT”“Gemini”通常只是家族或品牌，你在请求里需要填写控制台显示的具体模型名。

模型名要完整复制，包括版本号、后缀、大小写和分隔符。

::: warning
模型名请使用 Arqel 控制台中显示的具体名称。不要把模型家族名、其他平台别名或示例占位文字直接填进请求。
:::

## Arqel、工具和模型提供商的关系

Arqel 是接入层，不是 Cursor、Claude Code、Gemini CLI 或 Codex 本身，也不是模型提供商本身。

常见链路是：

```text
你的工具或代码 -> Arqel -> 模型提供商 -> Arqel -> 你的工具或代码
```

Arqel 负责统一入口、Key 管理、请求转发和用量记录。模型的具体能力、速度和可用性仍会受到上游模型服务影响。
