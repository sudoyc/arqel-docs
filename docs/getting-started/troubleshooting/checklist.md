# 接入检查清单

## API 测试前确认

- 可以打开 Arqel 控制台。
- 已创建 API Key。
- 已复制 Base URL。
- 已确认要使用的模型名。
- 终端可用，`curl --version` 或 `curl.exe --version` 能正常返回。
- 已准备一个简单问题用于测试。

## SDK / Agent 接入前确认

- 已确认 API Key、Base URL 和模型名来自 Arqel 控制台。
- 如果要使用 SDK 或命令行 Agent，再确认 `node -v`、`npm -v`、`git --version` 等对应工具命令正常返回。
- 已确认工具支持自定义 API Key 和 Base URL。
- 已确认工具当前产品入口支持需要的协议或 Provider 类型。
- 已确认当前工具里填写的是一个 Arqel 支持的具体模型名。
- 如果使用 CC Switch，已确认它管理的是你正在运行的那一个工具和用户目录。

## 接入后确认

- API 测试返回成功。
- 如果控制台提供用量或请求记录，能看到刚才的测试请求。
- 工具里的错误提示为空。
- 已核对请求时间、Key 名称和模型名。
- 如果工具能回复但控制台没有记录，先排查工具是否仍在使用默认 Provider。

## 上线前确认

- 生产环境和测试环境使用不同 Key。
- Key 没有写入前端代码。
- 浏览器前端不会直接调用 Arqel；请求经过你自己的后端。
- 后端会校验自己的用户身份，不是公开转发器。
- CORS / Origin 策略只允许可信前端来源。
- 已按用户、IP、项目或组织设置限流。
- 已限制请求体大小、最大输出长度和单次任务运行时间。
- 不允许普通用户传入任意 Base URL。
- 日志会隐藏 `Authorization`、Cookie、access token、API Key、用户隐私内容和客户文件路径。
- 异常状态有日志或提示，但不会把完整 secret 返回给前端。
- 已设置必要的额度、预算或用量提醒。
- 自动重试设置了最大次数，`401` / `403` 不自动重试。
- 请求超时、客户端断开和上游异常都有处理。
- 线上有暂停开关，可以快速停用某个 Key、用户、项目或代理入口。
- Agent 设置了最大轮次、运行时间或人工确认边界。
