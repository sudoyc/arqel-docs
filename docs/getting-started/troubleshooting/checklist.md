# 接入检查清单

## API 测试前确认

- 可以打开 Arqel 控制台。
- 已创建 API Key。
- 已复制 Base URL。
- 已确认要使用的模型名。
- 终端可用，`curl --version` 或 `curl.exe --version` 能正常返回。
- 已准备一个简单问题用于测试。

## SDK / Agent 接入前确认

- 第一条 API 请求已经返回成功。
- 如果要使用 SDK 或命令行 Agent，再确认 `node -v`、`npm -v`、`git --version` 等对应工具命令正常返回。
- 已确认工具支持自定义 API Key 和 Base URL。
- 已确认当前工具里填写的是一个 Arqel 支持的具体模型名。

## 接入后确认

- API 测试返回成功。
- 如果控制台提供用量或请求记录，能看到刚才的测试请求。
- 工具里的错误提示为空。
- 已核对请求时间、Key 名称和模型名。

## 上线前确认

- 生产环境和测试环境使用不同 Key。
- Key 没有写入前端代码。
- 异常状态有日志或提示。
- 已设置必要的额度或用量提醒。
